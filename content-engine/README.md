# LinkedIn Content Engine

Automated LinkedIn post generation with a human review gate. Posts are generated by Claude, reviewed by Tim, and scheduled via Buffer on merge.

---

## How it works

1. **Monday, Wednesday, Thursday, Friday at 8am AEST** — GitHub Actions picks the next topic from `topics.json`, generates two post variants via Claude, and opens a PR
2. **Tim reviews the PR** — reads both variants, edits if needed, sets frontmatter, merges
3. **On merge** — GitHub Actions schedules the approved variant to LinkedIn via Buffer

---

## Reviewing a post PR

Open the PR. You'll see both variants in the PR description and in the MDX file.

Edit the MDX file directly in the PR if needed, then update the frontmatter:

| Field | What to do |
|---|---|
| `approved_variant` | Set to `"A"`, `"B"`, or `"EDITED"` if you rewrote it |
| `schedule_time` | Set to when you want it to go live (ISO 8601 with AEST offset: `+10:00`) |
| `facebook` | Set to `true` if you want to repurpose to Facebook |
| `facebook_variant` | Add the Facebook version of the copy if `facebook: true` |

Merge the PR when ready. The publish workflow fires automatically.

## Managing the topic queue

Add topics to `topics.json`. The engine picks them up in order from the top.

Each topic has these fields:

```json
{
  "id": "021",
  "status": "queued",
  "tier": "unaware",
  "topic": "The topic in plain English",
  "seed_claim": "The core argument or insight — your POV, not generic",
  "audience_angle": "Who specifically is this post for and what do they believe right now",
  "cta_style": "none | soft | audit | direct",
  "notes": "Anything else useful for the AI — stats, angles to try, links"
}
```

**Tier values** (match your audience's buying temperature):
- `unaware` — 60% of market. Pattern interrupt. They don't know they have a problem.
- `problem_aware` — 20%. Education. They know email is leaking but haven't acted.
- `gathering_info` — 17%. Authority. They're actively researching options.
- `ready_to_buy` — 3%. Direct offer. Use sparingly — max 1 in every 8–10 posts.

**CTA style values:**
- `none` — pure value, no ask
- `soft` — "follow for more" or engagement question
- `audit` — link to free audit tool
- `direct` — book a call

**Recommended mix per fortnight (8 posts):**
2 unaware · 3 problem_aware · 2 gathering_info · 1 ready_to_buy

**Replenishing the queue:** aim for 4–6 weeks of topics at all times. One 20-minute sitting every 3–4 weeks to brainstorm 15–20 topics with seed claims is enough.

## Status values in topics.json

| Status | Meaning |
|---|---|
| `queued` | Ready to be picked up |
| `in_review` | PR is open — waiting on Tim |
| `published` | Merged and scheduled |
| `skipped` | Manually skipped |

## File structure

```
content-engine/
  topics.json          ← topic queue — manage this
  posted.json          ← auto-updated log of published posts
  prompts/
    system-prompt.txt  ← master copywriting instructions for Claude
    post-brief-template.md  ← per-post prompt template
  posts/
    drafts/            ← PRs land here
    published/         ← archived after publishing
  scripts/
    generate-post.js   ← called by generate workflow
    publish-post.js    ← called by publish workflow
```

## GitHub Secrets required

Add these in **Settings → Secrets → Actions**:

| Secret | Where to get it |
|---|---|
| `ANTHROPIC_API_KEY` | console.anthropic.com |
| `BUFFER_ACCESS_TOKEN` | Buffer → Settings → Apps → Access Token |
| `BUFFER_LINKEDIN_PROFILE_ID` | Buffer API or dashboard URL |
| `BUFFER_FACEBOOK_PAGE_ID` | Buffer API (optional) |

`GITHUB_TOKEN` is provided automatically by GitHub Actions — no setup needed.

---

## Buffer setup

1. Sign up at [buffer.com](https://buffer.com) — free plan supports 1 LinkedIn channel and 10 queued posts
2. Connect your LinkedIn personal profile
3. Go to **Settings → Apps** and copy your access token
4. Find your LinkedIn profile ID via the Buffer API: `GET https://api.bufferapp.com/1/profiles.json?access_token=YOUR_TOKEN`
5. Add both to GitHub secrets
